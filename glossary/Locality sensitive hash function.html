Locality sensitive hash function

Let h:ℝd→{1,…,m}h: \mathbb{R}^d \rightarrow \{1,…,m\} be a random hash function.

We call hh locality sensitive for similarity function s(𝐪,𝐲)s(\mathbf{q},\mathbf{y}) if Pr[h(𝐪)==h(𝐲)]\mathrm{Pr}[h(\mathbf{q})==h(\mathbf{y})] is

  • higher when 𝐪\mathbf{q} and 𝐲\mathbf{y} are more similar, i.e. s(𝐪,𝐲)s(\mathbf{q},\mathbf{y}) higher
  • lower when 𝐪\mathbf{q} and 𝐲\mathbf{y} are more dissimilar, i.e. s(𝐪,𝐲)s(\mathbf{q},\mathbf{y}) lower

Locality Sensitive Hash Family

For distances r1,r2r_1,r_2 with r1<r2r_1 < r_2, a family of hash functions ℋ:U→S\mathcal{H}: U \rightarrow S is (r1,r2,p1,p2)(r_1,r_2,p_1,p_2)-locality sensitive if for any x,y∈Ux,y \in U:

  1. If d(x,y)≤r1d(x,y)\leq r_1 then Prh∈ℋ[h(x)=h(y)]≥p1\mathrm{Pr}_{h\in\mathcal{H}}[h(x)=h(y)]\geq p_1
  2. If d(x,y)≥r2d(x,y)\geq r_2 then Prh∈ℋ[h(x)=h(y)]≤p2\mathrm{Pr}_{h\in\mathcal{H}}[h(x)=h(y)]\leq p_2

(For our purposes, we always have p2<p1p_2 < p_1. I.e. if two points are close together, they have a strictly higher probability of hashing to the same bucket than two points that are far apart. This property is ultimately what allows us to perform efficient near neighbor search.)

Tunable LSH

Full LSH scheme has two parameters to tune: tt tables and rr bands

  • effect of increasing number of tables tt: fewer false negatives, more false positives
  • effect of increasing number of bands rr: more false negatives, fewer false positives

s-curve tuning, collision probability vs Jaccard similarity

Crucial for Indyk and Motwani (1998) theorem.

Also see SimHash and MinHash (Broder, 1997).
